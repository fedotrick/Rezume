# Итоговый проект. Часть 1 — Система управления портфелями

Данный модуль представляет первую часть итогового проекта курса: проектирование базы данных и разработка ключевых хранимых процедур для системы управления портфелями ценных бумаг.

## Состав

- **`scripts/final_project_part1_schema.sql`** — создание таблиц, ограничений и индексов.
- **`scripts/final_project_part1_procedures.sql`** — хранемые процедуры для операций, аналитики и отчетности.
- **`scripts/final_project_part1_sample_data.sql`** — примеры данных (5 бумаг, 3 портфеля, сделки, котировки, операции).
- **`scripts/final_project_part1_examples.sql`** — демонстрационные запросы и примеры вызовов процедур.
- **`documentation/FINAL_PROJECT_PART1_DB_DESIGN.md`** — подробное описание схемы, логики процедур и примеров использования.

## Быстрый старт

1. Подключитесь к целевой базе данных SQL Server.
2. Выполните скрипт `final_project_part1_schema.sql` для создания структуры.
3. Выполните скрипт `final_project_part1_procedures.sql` для развёртывания процедур.
4. По необходимости выполните `final_project_part1_sample_data.sql`, чтобы наполнить базу тестовыми данными.
5. Откройте документацию в `documentation/FINAL_PROJECT_PART1_DB_DESIGN.md` для описания таблиц, процедур и примеров вызова.

## Минимальные требования

- Microsoft SQL Server 2017 или выше.
- Включённая поддержка работы с JSON (`OPENJSON`) для процедуры ребалансировки.

## Следующие шаги

- Интеграция процедур в прикладной слой.
- Создание дополнительных аналитических витрин и отчетов.
- Реализация автоматического заполнения `Audit_Log` на другие операции (например, триггеры при обновлении/удалении записей).
